## 1. 프로젝트 배경 및 목표

### 1.1 배경

일본어 학습 시장에는 Duolingo, WaniKani, Bunpro 등 다양한 앱이 존재하지만, 다음과 같은 갭이 있다:

- **통합 학습의 부재**: 대부분 단어/한자/문법 중 하나에 특화되어 있어 여러 앱을 병행해야 함
- **능동적 접근 필요**: 사용자가 앱을 직접 열어야 학습이 시작됨 → 습관 형성 어려움
- **실제 콘텐츠 부족**: 교과서적 문장이 아닌 뉴스, 칼럼 등 실제 일본어 콘텐츠 기반 학습 도구가 적음
- **메신저 네이티브 학습 도구 부재**: LINE/카카오톡/텔레그램에서 일본어를 체계적으로 학습할 수 있는 봇이 거의 없음

### 1.2 목표

1. **매일 지정 시간에 레벨 맞춤 일본어 콘텐츠를 메신저로 전달**
2. **메신저 내 명령어로 퀴즈(문장 만들기, 해석하기 등)를 풀 수 있는 인터랙티브 학습**
3. **웹 대시보드에서 레벨 관리, 학습 이력, 복습 기능 제공**

### 1.3 성공 지표

| 지표 | 목표 |
|------|------|
| DAU(일일 활성 사용자) | MVP 출시 후 3개월 내 100명 |
| 일일 퀴즈 완료율 | 70% 이상 |
| 주간 리텐션 | 60% 이상 |
| 평균 학습 시간 | 5-10분/일 |

## 2. 시장 조사 결과

### 2.1 경쟁 서비스 분석

| 서비스 | 특징 | 강점 | 약점 |
|--------|------|------|------|
| **Duolingo** | 게이미피케이션 기반 학습 | 습관 형성, 무료 | 문법 설명 부족, 자연스러운 일본어 아님 |
| **WaniKani** | 한자/어휘 SRS 학습 | 체계적 한자 학습 | 한자/어휘만, 문법/독해 없음 |
| **Bunpro** | 문법 SRS 학습 (N5-N1) | JLPT 문법 포인트 커버 | 문법만, 실제 콘텐츠 부족 |
| **Anki** | 범용 SRS 플래시카드 | 유연성 높음 | 셋업 복잡, UX 불편 |
| **Kotoba (Discord)** | 디스코드 일본어 봇 | 사전 검색, 퀴즈 | 체계적 학습 과정 없음, 푸시 학습 없음 |

### 2.2 차별화 포인트

1. **푸시 기반 학습**: 사용자가 앱을 열지 않아도 메신저로 콘텐츠 도착
2. **실제 콘텐츠 활용**: NHK News Web Easy 등 실제 일본어 뉴스/칼럼 기반
3. **통합 학습**: 어휘 + 문법 + 독해 + 작문을 하나의 플로우에서
4. **AI 기반 맞춤 퀴즈**: LLM을 활용한 레벨/관심사 맞춤 퀴즈 생성
5. **SRS 복습 시스템**: FSRS 알고리즘 기반 과학적 복습 스케줄링

## 3. 메신저 플랫폼 선정

### 3.1 플랫폼 비교

| 기준 | Telegram | Discord | LINE | KakaoTalk |
|------|----------|---------|------|-----------|
| 개발 난이도 | 쉬움 | 보통 | 보통 | 어려움 |
| 무료 한도 | **무제한** | **무제한** | 200-500건/월 | 제한적 |
| 리치 메시지 | 우수 | 양호 | 우수 | 양호 |
| 일본 사용자 | 적음 | 보통 | **최다 (9,700만)** | 없음 |
| 한국 사용자 | 성장중 | 성장중 | 없음 | **최다 (4,890만)** |
| 영문 문서 | 우수 | 양호 | 양호 | 미흡 |

### 3.2 추천: Telegram (MVP) → LINE/KakaoTalk (확장)

**MVP 단계에서 Telegram을 선택하는 이유:**
- 메시지 발송 완전 무료 (LINE은 월 200-500건 제한)
- Bot API가 가장 개발자 친화적이고 문서가 우수
- 인라인 키보드, 콜백 버튼 등 리치 인터랙션 지원
- 한국/일본 양쪽에서 사용 가능

**확장 단계:**
- 일본 타겟 → LINE Bot 추가
- 한국 타겟 → KakaoTalk Bot 추가

### 5.1 메신저 봇 기능

#### 5.1.1 데일리 콘텐츠 푸시

```
매일 [사용자 지정 시간]에 전송:
┌─────────────────────────────────────┐
│ 📚 오늘의 일본어 (N3 레벨)           │
│                                     │
│ 📰 NHK Easy 뉴스                    │
│ 「東京で新しい美術館がオープンしました」     │
│                                     │
│ 📝 오늘의 단어                       │
│ • 美術館 (びじゅつかん) - 미술관        │
│ • オープン - 개관, 오픈               │
│ • 展示 (てんじ) - 전시                │
│                                     │
│ 💡 오늘의 문법                       │
```

|-----------|------|-----------|

| **어휘 퀴즈** | 단어의 뜻 4지선다 | N5-N1 |
| **문법 빈칸** | 문장의 빈칸에 적절한 문법 선택 | N5-N1 |

| **어순 배열** | 섞인 단어를 올바른 순서로 배열 | N5-N3 |
| **뉴스 독해** | 기사 읽고 내용 파악 질문 | N3-N1 |
| **요약 작성** | 글을 읽고 일본어로 요약 | N2-N1 |

### 5.2 웹 대시보드 (Phase 2 - MVP 이후)

> **MVP에서는 웹 대시보드를 제외합니다.**
> 메신저 봇의 `/stats`, `/review` 명령어로 핵심 기능을 충분히 제공할 수 있으며,
> MVP 개발 기간을 3-4주로 단축하여 빠른 사용자 검증에 집중합니다.
> 웹은 사용자 확보 후 Phase 2에서 추가합니다.

**메신저로 대체하는 기능:**
- 레벨 설정/변경 → `/level` 명령어
- 학습 통계 → `/stats` 명령어 (텍스트 그래프)
- 복습 카드 → `/review` 명령어 (인라인 버튼)

**웹이 필요한 시점 (Phase 2):**
- 학습 캘린더 (잔디 히트맵) — 메신저로 표현 불가
- 상세 통계 차트 — 인터랙티브 시각화 필요
- 과거 콘텐츠 검색/아카이브 — 메신저 검색 한계
- 레벨 테스트 (배치 테스트) — 긴 폼 UI 필요

```

       ↓
```

```
├── id, user_id, quiz_id
├── user_answer, is_correct
├── time_spent_seconds
└── answered_at

ReviewCards (FSRS)
├── id, user_id
├── card_type (vocabulary/grammar/sentence)
├── card_ref_id
├── stability, difficulty
├── due_date, last_review
├── reps, lapses
└── state (new/learning/review/relearning)

DailyLogs
├── id, user_id, date
├── content_id, quizzes_completed
├── correct_count, total_count
└── study_minutes
```

## 8. 복습 알고리즘: FSRS

### 8.1 SM-2 대신 FSRS를 선택하는 이유

| 비교 항목 | SM-2 (1987) | FSRS (2022) |
|-----------|-------------|-------------|
| 복습 횟수 | 기준 | **20-30% 감소** |
| "Ease Hell" 문제 | 있음 | **없음** |
| 개인화 | 수동 | **자동 최적화** |
| 파라미터 설정 | 사용자 조정 필요 | **알고리즘이 학습** |

### 8.2 동작 방식

```
사용자 응답 → 품질 평가 (Again/Hard/Good/Easy)
    ↓
FSRS 계산 → 안정성(stability), 난이도(difficulty) 업데이트
    ↓
다음 복습일(due_date) 산출
    ↓
해당 날짜에 복습 카드로 제공
```

## 9. MVP 범위 및 로드맵

### 9.1 Phase 1: MVP (3-4주) — Telegram 봇 집중

| 주차 | 항목 | 상세 |
|------|------|------|
| 1주 | 프로젝트 셋업 + 봇 기본 | DB 스키마, `/start`, `/level`, `/time` 명령어, 사용자 등록 |
| 2주 | 콘텐츠 파이프라인 | NHK Easy 크롤러, Tatoeba 데이터 임포트, 레벨 분류 |
| 3주 | 데일리 푸시 + 퀴즈 | 스케줄러, 4종 기본 퀴즈 (읽기/어휘/문법/해석) |
| 4주 | 복습 + 통계 | FSRS 구현, `/review`, `/stats` 명령어 |

**MVP 포함 범위:**
- Telegram 봇 (1개 플랫폼) — 모든 학습 기능 메신저 내 완결
- N5-N3 레벨 (3개 레벨)
- 4종 퀴즈 (읽기, 어휘, 문법, 해석)
- FSRS 복습 시스템 (`/review`)
- 학습 통계 (`/stats` — 텍스트 기반 그래프)

**MVP 제외 범위:**
- 웹 대시보드 (Phase 2로 이동)
- LINE/KakaoTalk 봇
- N2-N1 레벨 콘텐츠
- 문장 만들기/요약 퀴즈 (AI 채점 필요)
- 소셜 기능 (랭킹, 친구)

### 9.2 Phase 2: 웹 대시보드 + 확장 (사용자 검증 후)

- **웹 대시보드** (Next.js)
